# opencode create-agent

Create a new AI agent with guided configuration and automatic prompt generation.

---

## 📌 Usage
```bash
opencode create-agent <agentName>
```

---
## 🧠 Overview
The `create-agent` command initializes a new agent by guiding you through a short interactive setup. It automatically generates:
* Agent configuration (YAML frontmatter)
* A structured system prompt
* A properly placed agent file (local or global)

---
## 🔄 Workflow
### 1. Select Scope
Choose where the agent should be stored:
* **Project (projectName)** → `.opencode/agents/<agentName>.md`
* **Global** → `~/.config/opencode/agents/<agentName>.md`

---

### 2. Configure Agent
#### Mode
Choose agent type:
* `primary` → standalone agent
* `subagent` → helper agent used by another agent
If `subagent` is selected, you will be asked to define its type (e.g. `code-reviewer`, `debugger`, `planner`).

---
#### Retry Steps
Define how many times the agent should retry a task before failing.
* Default: `5`
---

### 3. Agent Brainstorming (Smart Generation)

Instead of manually writing configuration, the command asks:

* What is the main goal of this agent?
* What tasks should it handle?
* What should it avoid?

Based on your answers, it automatically generates:

#### Description

* Max 200 characters
* Concise summary of agent purpose

#### System Prompt

Includes:

* Role definition
* Responsibilities
* Behavior rules
* Constraints
* Output format
* Failure handling

---

## 🧾 Generated File Structure

Each agent is stored as a Markdown file:

```md
---
description: <auto-generated description>

mode: primary | subagent

disable: false

hidden: true | false

model: anthropic/claude-sonnet-4-20250514

temperature: 0.1

steps: <user-defined or default 5>

permission:
  edit: deny
  bash:
    "*": deny
    "git diff": deny
    "git log*": deny
    "grep *": deny
    "ls": deny

  task:
    orchestrator: "allow"
    code-reviewer: "ask"
---

# Role
...

# Responsibilities
...

# Behavior Rules
...

# Constraints
...

# Output Format
...

# Failure Handling
...
```

---

## ⚙️ Defaults

| Setting     | Value                              |
| ----------- | ---------------------------------- |
| model       | anthropic/claude-sonnet-4-20250514 |
| temperature | 0.1                                |
| steps       | 5                                  |
| disable     | false                              |

---

## 🧩 Behavior Rules

* `primary` agents are visible and callable directly
* `subagent` agents are hidden and used internally
* Description is always auto-generated (≤200 chars)
* Permissions are locked by default for safety
* Prompt is generated based on structured interview

---

## 🚀 Example

```bash
opencode create-agent code-reviewer
```

Result:

* Prompts user for configuration
* Generates agent definition
* Saves to appropriate directory

---

## 💡 Tips

* Use clear, specific goals for better prompt generation
* Keep agent responsibilities focused
* Prefer `subagent` for specialized roles
* Adjust `steps` for more complex tasks
